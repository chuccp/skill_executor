## 可用工具

### 1. read_file - 读取源代码
阅读源文件以理解功能并生成文档。

### 2. list_directory - 查看项目结构
了解项目的文档目录和现有文档结构。

### 3. write_file - 创建文档文件
编写新的文档或更新现有文档。

### 4. glob - 查找相关文件
定位类型定义、配置文件等辅助文档编写。

## 文档标准

### Markdown 格式规范

```markdown
# 一级标题

## 二级标题

### 三级标题

正文内容应该：
- 简洁明了
- 使用主动语态
- 避免专业术语堆砌

**粗体** 用于强调重点
`行内代码` 用于技术名词

- 无序列表项
- 保持层次清晰

1. 有序列表
2. 用于步骤说明

```语言
代码块标注语言
```

| 表格 | 用于 |
|------|------|

```

## 写作原则

### 1. 面向读者
- 明确目标读者群体
- 使用读者熟悉的语言
- 回答读者关心的问题

### 2. 结构化
- 清晰的层次结构
- 一致的格式风格
- 便于扫读和检索

### 3. 示例驱动
- 提供实用的代码示例
- 展示常见使用场景
- 包含输入输出说明

### 4. 持续维护
- 标注文档最后更新时间
- 记录版本差异
- 及时修正过时内容

### API 文档模板
```markdown

## [端点名称]

\`METHOD /path/to/endpoint\`

| 参数 | 类型 | 必需 | 说明 |
|------|------|------|------|
| id | string | 是 | 资源 ID |
| limit | number | 否 | 返回数量限制 |

### 响应示例

\`\`\`json

| 状态码 | 说明 |
|--------|------|
